# Getting a repo ready for chalkbag

This doc has two parts:

- **Repo onboarding** — run once per repo to make it agent-ready.
- **Dev onboarding** — run once per machine to keep outputs fresh automatically.

The target model is:

- repo instructions live in a tracked `AGENTS.md`
- Claude reads a tracked `CLAUDE.md -> AGENTS.md` symlink when Claude is enabled
- shared skills, subagents, provider config, and permission config live under `.agents/`
- provider-native artifacts (`.claude/`, `.codex/`, `.opencode/`, `opencode.json`) are generated by chalkbag and gitignored

If you want an AI agent to do this work for you, point it at this file and tell it to follow the guide directly.

---

## Repo onboarding (run once per repo)

### 1. Scaffold the repo

```bash
cd ~/your-repo && chalkbag scaffold
```

This creates `.agents/` from the built-in template and an `AGENTS.md` stub if they are missing. It is idempotent — running it again will not overwrite your edits.

To also register the repo with the daemon and run the first build in one step:

```bash
cd ~/your-repo && chalkbag init
```

`chalkbag init` scaffolds, registers the repo in `repo` mode, and performs a synchronous first build. It prints the providers rendered and the paths written.

### 2. Edit the tracked root `AGENTS.md`

Replace the placeholder repo description, repo map, and working rules. Keep the file repo-specific. This file is your source of truth for agent instructions — not `.agents/`.

### 3. Edit `.agents/README.md`

Put chalkbag workflow and tooling notes here, not in root `AGENTS.md`.

### 4. Review `.agents/providers.yaml`

This file is required for schema validation. It declares which providers are active for this repo:

```yaml
providers:
  claude:
    enabled: true
  codex:
    enabled: true
  opencode:
    enabled: true
```

The `enabled` field controls schema validation but not which providers `chalkbag build` renders — pass `--provider <ids>` to the build command to restrict rendering.

### 5. Import shared skills (optional)

Add a `.agents/config.yaml` to reference external sources:

```yaml
imports:
  - source: github:<org>/<repo>
    ref: v1.2.0
    path: skills/oncall
```

Local files in `.agents/` always win over imported files. The `ref` can be a branch, tag, or commit SHA. See [agents-spec.md — External imports](./agents-spec.md#7-external-imports) for details.

### 6. Adjust optional inputs if needed

- Edit `.agents/permissions.yaml` if the repo needs provider-native permission rendering.
- Add shared skills under `.agents/skills/<skill-name>/`.
- Add shared subagents under `.agents/subagents/<name>.md`.

### 7. Extend the repo's `.gitignore`

Append this snippet to your repo's `.gitignore`:

```gitignore
# chalkbag — generated provider-native artifacts
.claude/
.codex/
.opencode/
opencode.json
```

Do not ignore the tracked root `AGENTS.md` or the tracked `CLAUDE.md` symlink.

### 8. Render once

```bash
cd ~/your-repo && chalkbag validate && chalkbag build --yes
```

`validate` checks the merged source tree against the schema, including any resolved imports.
`build` writes the generated provider-native outputs and updates `.agents/.state.json`.

Use `--provider <ids>` only as a one-off override:

```bash
cd ~/your-repo && chalkbag build --provider claude,codex --yes
```

If you omit `--provider`, chalkbag falls back to the last rendered provider set, or renders all providers if there is no prior state.

### 9. Create the root Claude bridge if Claude is enabled

```bash
cd ~/your-repo && ln -sf AGENTS.md CLAUDE.md
```

Claude reads `CLAUDE.md`; Codex and the rest of the agent ecosystem read `AGENTS.md` directly. Commit this symlink.

For scoped directories that need Claude to see their guidance:

```bash
cd ~/your-repo/packages/server && ln -sf AGENTS.md CLAUDE.md
```

### 10. Verify the result

```bash
cd ~/your-repo && chalkbag doctor
```

`chalkbag doctor` reports the daemon status, heartbeat freshness, config paths, and whether the rendered outputs are up to date.

## Dev onboarding (run once per machine)

### Install the daemon on macOS

```bash
chalkbag daemon install
chalkbag daemon status
```

The daemon watches all registered paths and rebuilds on file change. After installing, register your repos:

```bash
chalkbag register ~/your-repo
```

To register a parent directory that contains many repos (chalkbag watches one level deep for repos with `.agents/`):

```bash
chalkbag register --parent ~/Documents/Projects
```

Or use the alias:

```bash
chalkbag register-group ~/Documents/Projects
```

Check what is registered:

```bash
chalkbag paths
```

### Linux fallback / inline watcher

On Linux, or if you prefer not to install the daemon:

```bash
cd ~/your-repo && chalkbag watch
```

`chalkbag watch` is an inline watcher that stays in the foreground. It is equivalent to the daemon's per-repo watcher but exits when you close the terminal.

## Day-to-day workflow

- Edit `.agents/` files and root `AGENTS.md`. Never hand-edit `.claude/`, `.codex/`, `.opencode/`, or `opencode.json` — the next render overwrites them.
- Keep `CLAUDE.md` as a symlink. If it becomes a regular file, recreate it:

  ```bash
  cd ~/your-repo && ln -sf AGENTS.md CLAUDE.md
  ```

- Run `chalkbag validate` before `chalkbag build --yes` when making structural changes to `.agents/`.
- The daemon or `chalkbag watch` keeps outputs fresh. Git hooks are the fallback: if the daemon heartbeat is stale, commits still trigger an inline rebuild.
- To temporarily pause daemon rebuilds for a repo (for example, during a long refactor):

  ```bash
  chalkbag daemon pause
  ```

  Resume when ready:

  ```bash
  chalkbag daemon resume
  ```

## Troubleshooting

See [troubleshooting.md](./troubleshooting.md) for a full list of symptoms and fixes.

Quick reference:

| Symptom | Fix |
|---|---|
| `chalkbag doctor` reports artifacts not rendered | `chalkbag build --yes` |
| `chalkbag doctor` reports daemon not installed | `chalkbag daemon install` on macOS, or use `chalkbag watch` |
| Generated directories keep reappearing in `git status` | Append the `.gitignore` snippet from step 7 above |
| `CLAUDE.md` was committed as a regular file | `ln -sf AGENTS.md CLAUDE.md` and commit the symlink |
| `.agents/` edits do not propagate | `chalkbag daemon status`; reload or resume if needed |
